# knot

이 vault의 모든 규약은 `schema.md`에 있다. 어떤 작업이든 `schema.md`를 먼저 정독하라.
워크플로는 `prompts/`의 해당 파일(ingest·query·lint)을 따른다.

## 카파시 가이드라인 (AI 코딩 운영 규칙)

안드레 카파시의 프롬프트 엔지니어링 철학을 기반으로, AI가 코딩할 때 지켜야 할 운영 규칙이다.

### 핵심 원칙

1. **역할과 맥락을 명확하게 정의할 것**
   - AI에게 역할을 부여할 때 구체적으로 "무엇을 하는 전문가인지" 명시
   - 프로젝트의 배경, 목적, 제약조건을 충분히 설명
   - 예시: "너는 10년 경력의 백엔드 개발자로서, 이 프로젝트의 아키텍처를 리뷰하는 역할이다"

2. **구체적인 지시사항을 제공할 것**
   - 모호한 지시보다는 "어떤 파일을, 어디를, 어떻게 고칠 것인지" 구체적으로 명시
   - "이 함수의 에러 처리를 개선해줘"보다는 "utils.py의 parse_json 함수에서 JSONDecodeError 발생 시 로깅 후 None 반환하도록 고쳐줘"

3. **예시를 제공할 것**
   - 원하는 출력 형식이 있으면 예시를 보여줘
   - 좋은 예와 나쁜 예를 모두 보여주면 더 정확한 결과를 얻을 수 있다
   - 예시: "이렇게 고쳐줘: [좋은 예] / 이렇게 하지 마: [나쁜 예]"

4. **단계별로 나누어 설명할 것**
   - 복잡한 작업은 작은 단계로 나누어 지시
   - 한 번에 모든 것을 요청하지 말고, 단계별로 진행
   - 예시: "먼저 파일 구조를 파악하고, 그 다음 에러가 나는 부분을 찾고, 마지막으로 수정해"

5. **출력 형식을 명확하게 정의할 것**
   - 코드만 필요한지, 설명도 필요한지 명시
   - 주석 스타일, 파일 분할 방식 등 구체적으로 지정
   - 예시: "코드에는 주석을 달지 마. 대신 수정 이유를 별도로 설명해줘"

6. **제약조건을 설정할 것**
   - "절대 하지 말아야 할 것"을 명확히 지정
   - 기존 코드 스타일, 라이브러리 버전, 성능 제약 등
   - 예시: "기존 함수 시그니처는 절대 바꾸지 마. 새 함수를 만들어서 연결해"

### MiMo 적용 방법

- mimo_proxy.py의 시스템 프롬프트에 이 규칙들을 포함
- 코딩 작업 시 이 규칙들을 자동으로 따르도록 설정
- 새로운 스킬을 만들 때 이 가이드라인을 기반으로 설계
