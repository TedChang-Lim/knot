# 에이전트 표준 전쟁과 멀티 소켯 비유

## 핵심 아이디어

**마스터님(임창식)의 통찰.** 2026년 6월 23일 해나(Hermes Agent)와의 대화에서 나온 아이디어.

> "표준이 꼭 한 개일 필요가 없게 되지 않을까 싶네."

MCP(Anthropic), ACP(Zed), A2A(Google), ARD 등 여러 에이전트 통신 표준이 난립하는 상황에서, **브라우저가 여러 프로토콜(HTTP/HTTPS/WebSocket/WebRTC)을 동시에 지원하는 것처럼** 각 에이전트도 여러 표준용 **소켓**을 장착하게 될 것이라는 비유.

## 배경: 에이전트 표준 전쟁

| 표준 | 주체 | 용도 | 현황 |
|------|------|------|------|
| **MCP** (Model Context Protocol) | Anthropic | AI ↔ 도구/데이터 소스 | Hermes Agent 공식 지원 |
| **ACP** (Agent Client Protocol) | Zed Industries | 에이전트 ↔ 에디터 | Claude Code, Gemini CLI 연결 |
| **A2A** (Agent-to-Agent) | Google | 에이전트 ↔ 에이전트 직접 통신 | 등장 초기 |
| **ARD** (Agent Runtime Discovery) | (논의 중) | 에이전트 런타임 발견 | 표준화 진행 중 |

## 마스터님의 비유: 멀티 소켓

> "많은 에이전트들이 하나의 그 표준 — 그러니까 MCP를 지원하면 ARD도 지원할 거고, 여러 가지를 지원해서 서로 소통할 수 있게... **소켓을 여러 개 장착**해 가지고 '아 얘 이쪽으로 소통 안 되면 이 소켓으로는 소통되네?' 하면서 되지 않을까?"

즉:
- **표준이 하나로 통일될 필요는 없다**
- 각 플랫폼(Anthropic, Google, Zed, OpenAI)은 각자의 표준을 계속 밀겠지만
- **에이전트 입장에서는 여러 소켓을 장착**해서 상대방이 지원하는 프로토콜을 자동 탐지(fallback)하면 됨
- 사용자는 "Chrome 하나면 모든 웹사이트가 보인다"처럼 **하나의 에이전트로 모든 에이전트와 소통 가능**

## 함의

### 에이전트 간 통신의 미래
```
현재: to-hena.md / to-ag.md / 모아(knot) → 파일 기반 우회 통신
미래: 해나(MCP+ACP+A2A+...) → 직접 메시지 라우팅
```

### 파일 기반 통신의 운명
- 표준이 완성되면 **모아(knot)와 to-*.md 파일은 불필요**해질 수 있음
- 에이전트가 필요한 지식을 다른 에이전트에게 직접 물어보는 시대
- 마스터님이 중개자 역할을 할 필요도 없어짐

### BUT 현실
- 표준 전쟁이 끝나고 상호운용성이 확보되기까지는 시간 필요
- 그때까지는 **현재의 파일 기반 통신이 가장 확실한 방법**
- Hermes의 `delegate_task`는 같은 Hermes 인스턴스 내에서만 동작
