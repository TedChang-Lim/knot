## 서브도메인 설정 및 HTTPS 연동 과정 (개발 가이드)

향후 미모나 헤나가 다른 서브도메인을 추가하거나 유사한 작업을 할 때 참고할 수 있도록 전체 연동 과정을 기록합니다.

1. **DNS 레코드 등록 (도메인클럽)**
   - 도메인클럽 DNS 설정 페이지에서 각 서브도메인(`s`, `w`, `j`)에 대해 **CNAME 레코드**를 생성하고, 목적지를 `tedchang-lim.github.io.`로 등록합니다. (마지막 온점 필수)
2. **저장소 루트에 CNAME 파일 추가 (가장 중요)**
   - 각 GitHub 저장소의 `main` 브랜치 루트 디렉토리에 확장자 없는 `CNAME` 파일을 생성하고 내용물로 해당 서브도메인 주소(예: `s.kacec.kr`)만 한 줄 적어 커밋/푸시합니다.
   - *주의:* 이 파일이 없으면 GitHub Pages에서 도메인을 저장해도 수동으로 초기화되는 버그가 발생합니다.
3. **GitHub Repository Pages 설정**
   - 각 저장소의 **Settings → Pages** 메뉴로 이동합니다.
   - **Custom domain** 칸에 서브도메인 주소를 입력하고 **Save**를 누릅니다.
4. **DNS 검사 및 HTTPS 강제 활성화**
   - 저장 후 DNS Check가 시작됩니다. 전파 시간에 따라 몇 분에서 길게는 수십 분 소요될 수 있습니다.
   - `✓ DNS check successful` 메시지가 뜨고, TLS 인증서 발급이 완료되면 아래의 **Enforce HTTPS** 체크박스가 활성화됩니다.
   - 반드시 **Enforce HTTPS**를 선택해 주어야 주소 표시줄에 자물쇠 아이콘(보안 연결)이 생깁니다.
